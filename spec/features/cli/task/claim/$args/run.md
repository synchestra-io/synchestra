# --run

Unique identifier for the agent run claiming a task.

| Detail | Value |
|---|---|
| Type | String |
| Required | Yes |
| Default | — |

## Supported by

[`task claim`](../README.md)

## Description

Identifies which agent run is claiming the task. This value is recorded in the task metadata for auditability and debugging — it allows tracing which agent instance worked on which task.

The run ID is typically provided by the orchestrator or generated by the agent framework. It should be unique across concurrent runs.

## Examples

```bash
synchestra task claim --project synchestra --task implement-cli \
  --run 4821 --model sonnet

synchestra task claim --project my-service --task fix-bug \
  --run agent-session-abc123 --model opus
```
