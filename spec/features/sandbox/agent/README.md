The gRPC service specification for container-side agent (`SandboxAgent`). Defines all RPCs, message types, and communication semantics.

**Service Location**: Container listens on `/var/run/synchestra-{project_id}.sock` (Unix socket)

**Transport**: 
- Recommended: Unix socket (no TLS needed, file permissions enforce access)
- Fallback: TCP localhost with mTLS (if needed for future remote orchestration)

**Serialization**: Protobuf 3

## Service Definition

```protobuf
syntax = "proto3";

package synchestra.sandbox.v1;

import "google/protobuf/empty.proto";
import "google/protobuf/timestamp.proto";
import "google/protobuf/struct.proto";

service SandboxAgent {
  // Execute a command in an isolated session
  rpc ExecuteCommand(CommandRequest) returns (stream CommandOutput) {}
  
  // Get container status and active sessions
  rpc GetStatus(StatusRequest) returns (StatusResponse) {}
  
  // List active sessions
  rpc ListSessions(ListSessionsRequest) returns (SessionList) {}
  
  // Stream logs from a session (real-time)
  rpc StreamLogs(StreamLogsRequest) returns (stream LogEntry) {}
  
  // Store or update a credential (user provides token, container encrypts)
  rpc StoreCredential(StoreCredentialRequest) returns (StoreCredentialResponse) {}
  
  // Get task state from local .synchestra/ repo
  rpc GetTaskState(GetTaskStateRequest) returns (TaskStateResponse) {}
  
  // Retrieve stored credential (decrypted, for command execution)
  rpc GetCredential(GetCredentialRequest) returns (GetCredentialResponse) {}
  
  // Health check (liveness probe)
  rpc Ping(google.protobuf.Empty) returns (PingResponse) {}
}
```

## Messages

### ExecuteCommand

Execute a shell command in an isolated session workspace.

```protobuf
message CommandRequest {
  string session_id = 1;
  // Unique identifier for this execution session.
  // Generated by host; used for isolation and logging.
  // Format: "{user_id}-{timestamp}-{random_suffix}"
  
  string user_id = 2;
  // User identifier (from auth system).
  // Used for: session directory ownership, credential access validation, audit logging.
  
  repeated string command = 3;
  // Command to execute.
  // Format: ["/bin/bash", "-c", "git clone https://..."]
  // Similar to execve argv[].
  
  string working_dir = 4;
  // Working directory (relative to session directory).
  // Default: "working" (resolves to /workspace/{project_id}/sessions/{session_id}/working/)
  // Validates against path traversal (must be within session directory).
  
  map<string, string> env_vars = 5;
  // Environment variables to inject (in addition to inherited container env).
  // Keys are validated (no PATH override, SYNCHESTRA_* reserved).
  // Secrets must NOT be passed here; use StoreCredential + GetCredential flow.
  
  int32 timeout_seconds = 6;
  // Command timeout (wall-clock seconds).
  // If exceeded, container kills subprocess and returns TIMEOUT exit code.
  // Default: 1800 (30 minutes).
  // Range: [1, 86400] (1 second to 24 hours).
  
  bool stream_output = 7;
  // If true, output is streamed in real-time (multiple CommandOutput messages).
  // If false, output is buffered and returned in single message on completion.
  // Recommended: true (better UX, stream to user immediately).
}

message CommandOutput {
  string session_id = 1;
  // Echo session_id (correlates output to request).
  
  bytes stdout = 2;
  // Chunk of stdout (if stream_output=true, multiple chunks).
  // If stream_output=false, contains all stdout on completion.
  
  bytes stderr = 3;
  // Chunk of stderr (same chunking as stdout).
  
  int32 exit_code = 4;
  // Exit code of process.
  // 0 = success
  // 1-255 = process exit code
  // -1 = killed by signal
  // -2 = security violation (path traversal, permission denied)
  // 124 = timeout (SIGTERM timeout)
  // 137 = OOM killed (SIGKILL from kernel)
  // 139 = segmentation fault
  // Only populated on final message (completed=true).
  
  bool completed = 5;
  // True on final message (command finished, no more output).
  // Consumers can detect completion and stop reading stream.
  
  string error_message = 6;
  // Error message if command failed to start or other system error.
  // Empty on success.
  // Only populated on final message if error occurred.
}
```

**Behavior:**
1. Host sends `CommandRequest` to container via gRPC
2. Container creates `/workspace/{project_id}/sessions/{session_id}/` directory tree
3. Container sets working_dir, environment, resource limits (cgroups)
4. Container executes command via subprocess (not shell, unless shell is in command)
5. Container captures stdout/stderr
6. If `stream_output=true`: stream chunks as they arrive
7. If `stream_output=false`: buffer all output, send on completion
8. On timeout: send SIGTERM, wait 5s, send SIGKILL if still alive
9. Return final message with exit_code and completed=true
10. On stream completion, container may clean up session directory (after retention period)

**Error Cases:**
- `PermissionDenied`: User does not have access to project
- `InvalidArgument`: Command empty, path traversal detected, timeout out of range
- `ResourceExhausted`: Session limit reached for container, or disk quota exceeded
- `Internal`: Container error (cannot write to filesystem, etc.)

### GetStatus

Get current container status and resource usage.

```protobuf
message StatusRequest {
  // Empty request (no parameters).
}

// Container lifecycle status
enum ContainerStatus {
  CONTAINER_STATUS_UNSPECIFIED = 0;
  CONTAINER_STATUS_RUNNING = 1;
  CONTAINER_STATUS_PAUSED = 2;
  CONTAINER_STATUS_STOPPED = 3;
  CONTAINER_STATUS_FAILED = 4;
}

// Session lifecycle status
enum SessionStatus {
  SESSION_STATUS_UNSPECIFIED = 0;
  SESSION_STATUS_PENDING = 1;
  SESSION_STATUS_RUNNING = 2;
  SESSION_STATUS_COMPLETED = 3;
  SESSION_STATUS_FAILED = 4;
}

message StatusResponse {
  string container_id = 1;
  // Docker container ID (for debugging).
  
  ContainerStatus status = 2;
  // Container lifecycle status.
  
  int64 uptime_seconds = 3;
  // Seconds since container started (wall-clock).
  
  ContainerResources resources = 4;
  // Current resource usage.
  
  repeated SessionInfo active_sessions = 5;
  // List of currently active (non-completed) sessions.
}

message ContainerResources {
  int64 memory_used_mb = 1;
  // Memory usage (resident set size, MB).
  
  int64 memory_limit_mb = 2;
  // Memory limit (from Docker config).
  
  float cpu_usage_percent = 3;
  // CPU usage (0-100%, can exceed 100% on multi-core).
  
  float cpu_limit = 4;
  // CPU limit (cores).
  
  int64 disk_used_gb = 5;
  // Disk usage of /workspace/ (GB).
  
  int64 disk_limit_gb = 6;
  // Disk quota (from config).
  
  int32 active_sessions_count = 7;
  // Number of currently running sessions.
}

message SessionInfo {
  string session_id = 1;
  string user_id = 2;
  SessionStatus status = 3;   // Session lifecycle status
  int32 command_count = 4;     // Number of commands executed in this session
  int64 memory_used_mb = 5;    // Memory used by session subprocess
  int64 runtime_seconds = 6;   // Seconds since session started
  string working_dir = 7;      // Path to session working directory
}
```

### ListSessions

List sessions (optionally filtered by user_id).

```protobuf
message ListSessionsRequest {
  string user_id = 1;
  // Optional: filter to sessions of specific user.
  // If empty, returns all sessions.
  
  string status_filter = 2;
  // Optional: filter by status ("running", "completed", etc.).
  // If empty, returns all statuses.
  
  int32 page_size = 4;
  // Max results per page. Default: 50
  
  string page_token = 5;
  // Token for next page (from previous response)
}

message SessionList {
  repeated SessionInfo sessions = 1;
  int32 total_count = 2;       // Total sessions matching filter
  string next_page_token = 3;  // Token for next page, empty if last page
}
```

### StreamLogs

Stream real-time logs from a session (for live output display).

```protobuf
message StreamLogsRequest {
  string session_id = 1;
  // Session ID to stream logs from.
  
  bool follow = 2;
  // If true, keep stream open after final output (follow mode).
  // If false, close stream once session completes.
  // Recommended: true (better UX).
  
  int32 tail = 3;
  // Return only last N log lines (for reconnection after temporary disconnect).
  // If 0, return all logs. If 100, return last 100 lines.
}

message LogEntry {
  google.protobuf.Timestamp timestamp = 1;
  
  string stream = 2;
  // "stdout" or "stderr".
  
  bytes data = 3;
  // Log line data (may contain newlines for multi-line output).
}
```

### StoreCredential

Store or update an encrypted credential in the container's local vault.
Existing credentials with the same identifier are overwritten silently.

```protobuf
message StoreCredentialRequest {
  string credential_type = 1;
  // Type of credential: "git_token", "ssh_key", "api_key", "aws_access_key", etc.
  
  string credential_value = 2;
  // The actual secret (sent over TLS if using TCP, or Unix socket if local).
  // Container will encrypt this immediately upon receipt.
  // WARNING: Should never be logged or echoed back.
  
  string identifier = 3;
  // Human-readable identifier for this credential.
  // Examples: "github-prod", "deploy-key-1", "aws-dev".
  // Used for lookup and display purposes.
  
  string description = 4;
  // Optional description for this credential.
  // Examples: "GitHub personal access token for pushing to main".
  
  int64 expires_at_unix = 5;
  // Optional: Unix timestamp when credential expires.
  // If 0 or omitted, credential has no expiry.
  // Container may warn user before expiry or refuse expired credentials.
}

message StoreCredentialResponse {
  bool success = 1;
  string message = 2;
  // Success message or error details.
  // Example: "Credential 'github-prod' stored and encrypted."
  // If a credential with the same identifier already exists, it is overwritten.
  // Example (overwrite): "Credential 'github-prod' updated and encrypted."
  
  // NOTE: Response NEVER contains the credential value or decryption key.
}
```

**Behavior:**
1. Container receives credential_value
2. Container encrypts: `AES256-GCM(credential_value, encryption_key, nonce)`
3. Container stores in `/workspace/{project_id}/.secure/credentials.enc`
4. Returns success
5. Never logs credential_value or encryption_key

### GetCredential

Retrieve and decrypt a stored credential (for use in command execution).

```protobuf
message GetCredentialRequest {
  string identifier = 1;
  // Identifier of credential to retrieve (must match StoreCredential identifier).
  
  string user_id = 2;
  // User requesting credential (for audit logging).
  // Container validates user has permission to use this credential.
}

message GetCredentialResponse {
  string credential_type = 1;
  string credential_value = 2;
  // Decrypted credential value.
  // WARNING: Should not be logged or stored by client.
  
  bool found = 3;
  bool expired = 4;
  // If expired=true, credential_value is not populated; client should reject.
}
```

**Behavior:**
1. Container validates user_id has access to project (basic check; could add per-credential ACL)
2. Container decrypts credential from vault
3. Returns decrypted value
4. Logs access attempt (not the value): `user={user_id} identifier={identifier} type={type}`

## Error Handling

### gRPC Status Codes

Standard gRPC status codes used in responses:

- `OK (0)`: Success
- `CANCELLED (1)`: Request cancelled by client
- `UNKNOWN (2)`: Unknown error
- `INVALID_ARGUMENT (3)`: Invalid request (empty command, timeout out of range, etc.)
- `DEADLINE_EXCEEDED (4)`: Timeout (command exceeded timeout_seconds)
- `NOT_FOUND (5)`: Session/credential/state not found
- `PERMISSION_DENIED (7)`: User does not have access (project, credential)
- `RESOURCE_EXHAUSTED (8)`: Session limit reached, disk quota exceeded, memory limit
- `FAILED_PRECONDITION (9)`: Container exists but has not completed initialization (state repo not cloned, agent not ready). Permanent until initialization completes — client should NOT retry immediately.
- `ABORTED (10)`: Command aborted (e.g., killed by signal)
- `INTERNAL (13)`: Container internal error (filesystem, subprocess error)
- `UNAVAILABLE (14)`: Container is temporarily unreachable (paused, restarting, network issue). Transient — client SHOULD retry with exponential backoff.
- `UNAUTHENTICATED (16)`: Authentication required (missing user_id header)

**Example Error Response:**
```
Status: RESOURCE_EXHAUSTED
Message: "Session memory limit (100MB) exceeded. Process killed."
Details: { "limit_mb": 100, "used_mb": 102, "session_id": "user1-20260315-abc123" }
```

### gRPC Status Code → exit_code Mapping

| gRPC Status Code | exit_code | Scenario |
|---|---|---|
| `RESOURCE_EXHAUSTED (8)` | 137 | Command killed by OOM (SIGKILL from kernel) |
| `PERMISSION_DENIED (7)` | _(none)_ | Command not started; gRPC error only |
| `DEADLINE_EXCEEDED (4)` | 124 | Command killed after timeout_seconds elapsed |

## Authentication & Authorization

### User Identification

Each RPC request includes user_id (from gRPC metadata headers):
```
user-id: {user_id}
```

### Authorization

Container validates:
1. User has access to project (basic check)
2. For StoreCredential: user can store credentials (usually all users in project can)
3. For GetCredential: user can read specific credential (add ACL if needed)
4. For ExecuteCommand: user can execute commands (check project access level)

### Security Notes

- Do NOT pass credentials as RPC parameters (use StoreCredential → GetCredential flow)
- Do NOT log credential values in any context
- All sensitive operations (credential access, task state mutations) should log user_id for audit trail

## Backwards Compatibility

### Versioning

- Package: `synchestra.sandbox.v1` (supports future v2, v3, etc.)
- Fields use explicit field numbers (allows adding fields without breaking)
- New optional fields added with high numbers (not affecting existing parsers)
- Deprecated fields marked with `deprecated = true` option

### Migration Path

If future changes require breaking changes:
1. Deploy v2 service alongside v1
2. Host gradually routes new requests to v2
3. Drain v1 requests, then remove v1
4. v1 clients auto-fallback or update to v2

## Performance Characteristics

### Expected Latencies

- `Ping()`: < 1ms
- `ExecuteCommand()` (first message): < 100ms
- `GetStatus()`: < 50ms
- `StoreCredential()`: < 200ms (includes AES256 encryption)
- Command output streaming: Real-time (< 100ms per chunk)

### Throughput

- Concurrent streams: 100+ (gRPC multiplexing)
- Message size: Typical 1-10KB per chunk (stdout/stderr)
- Large commands: Up to 4GB per message (protobuf default max)
